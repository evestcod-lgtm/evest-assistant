# Evest Assistant

Голосовой Android-ассистент на Kotlin. Имя ассистента — **Евестевень**.
Активация по фразе «евестевень», офлайн распознавание речи и wake-word,
умные ответы через Groq LLM, реальные системные действия (звонки,
приложения, навигация, будильники, фонарик и т.д.), работа в фоне при
выключенном/заблокированном экране, анимированный «глаз»-индикатор как у Siri.

---

## 1. Архитектура

**Стек:** Kotlin, Jetpack Compose (UI), Android Foreground Service (фон),
OkHttp (Groq API), Vosk (офлайн распознавание речи И wake-word — одна общая
модель на обе задачи), Android TextToSpeech (голос), EncryptedSharedPreferences
(хранение ключей).

Почему не Python/Kivy, как в исходном запросе: связка
Kivy + Buildozer + PyJNIus на практике крайне нестабильна для проектов с
foreground-сервисами и системными Intent'ами — такие сборки часто падают на
рантайме уже после того, как APK успешно собрался. Нативный Kotlin даёт:
официальную поддержку от Google, стабильные foreground-service API, прямой
доступ ко всем Intent/Manager, и предсказуемую сборку через Gradle в GitHub
Actions.

### Модули

| Модуль | Файл | Роль |
|---|---|---|

| STT команд | `stt/OfflineSpeechRecognizer.kt` | Распознаёт команду в текст офлайн после активации |
| NLU (правила) | `nlu/RuleBasedParser.kt` | Быстрый разбор типовых команд без сети |
| NLU (LLM) | `nlu/LlmIntentClassifier.kt` | Разбор через Groq, если правила не сработали |
| LLM-диалог | `llm/GroqClient.kt` | Обычный разговор / умные ответы |
| Действия | `actions/ActionExecutor.kt` | Реальные системные действия Android |
| TTS | `tts/VoiceSpeaker.kt` | Озвучивание ответа |

| Хранилище | `data/SettingsStore.kt` | Зашифрованные API-ключи, память, история |

### Как работает wake word

**Почему именно так:** изначально рассматривался Porcupine (Picovoice) —
специализированная нейросеть под одно слово, но их бесплатная регистрация
требует «корпоративный» email и на практике недоступна многим личным
пользователям. Альтернатива openWakeWord (обучение своей модели) уперлась в
постоянно ломающиеся версии зависимостей в Google Colab. Поэтому в этой
сборке wake-word работает **через ту же офлайн-модель Vosk**, что и
распознавание команд — она уже была нужна проекту в любом случае, и
дополнительных моделей/ключей/обучения не требует вообще.

Приложение постоянно слушает микрофон в фоне (см. `WakeWordDetector.kt`),
Vosk расшифровывает звук в текст небольшими фрагментами, и каждый фрагмент
проверяется на совпадение с «евестевень» (с запасом на небольшие ошибки
распознавания — «евестевень», «евестовень», «евистевень» и т.п.).

**Оптимизация батареи (важно):** прежде чем отдавать звук в Vosk, приложение
считает громкость (RMS-энергию) короткого фрагмента. Если фрагмент — тишина
(а это большую часть времени фонового прослушивания), Vosk вообще не
запускается — тяжёлая работа распознавания просто пропускается. Это резко
снижает расход батареи по сравнению с постоянным гоном полного
распознавания речи 24/7.

**Подавление шума:** к аудиопотоку подключаются встроенные в Android
аппаратные/программные эффекты `NoiseSuppressor` и `AcousticEchoCanceler`
(если доступны на конкретном устройстве) — это не требует отдельной
библиотеки и работает «из коробки» на большинстве современных телефонов.

**Честный компромисс:** точность распознавания одного конкретного
изобретённого слова через модель общего назначения (Vosk) ниже, чем у
специализированной wake-word модели (Porcupine/openWakeWord). Возможны
как случаи, когда «евестевень» не распознаётся с первого раза, так и редкие
случаи ложного срабатывания на похожие по звучанию слова. Для личного
использования это приемлемо; если точность окажется недостаточной, в
настройках приложения можно скорректировать список принимаемых вариантов
слова в `WakeWordDetector.kt` (`wakePhraseVariants`).

### Как работает распознавание команды

После срабатывания wake word включается тот же движок Vosk в режиме
непрерывного распознавания полной фразы (а не просто детекции ключевого
слова) — это стандартная офлайн-модель `vosk-model-small-ru-0.22` (~45 МБ).
Работает без интернета. Точность ниже, чем у облачного Google STT, особенно
в шуме — это осознанный компромисс в пользу приватности и офлайн-работы.

### Как используется Groq

Два сценария:
1. **Классификация неизвестных команд** — если правила (`RuleBasedParser`)
   не распознали фразу, она уходит в Groq с промптом-классификатором,
   который возвращает JSON с intent и слотами.
2. **Обычный разговор** — если это не команда, Groq отвечает как чат-бот,
   ответ озвучивается голосом.

Ключ хранится в `EncryptedSharedPreferences` (AES-256, привязано к Android
Keystore устройства), меняется в настройках без пересборки APK.

### Как работает TTS

Стандартный `android.speech.tts.TextToSpeech` — русский голос по умолчанию,
настройка скорости/тона в приложении, выбор конкретного голоса из
установленных на устройстве.

### Как приложение работает в фоне

`AssistantForegroundService` — foreground service с постоянным
уведомлением (обязательно по правилам Android), типом `microphone`
(требование Android 14+), и partial `WakeLock`, чтобы CPU не засыпал
глубоко и wake-word слушался даже при заблокированном экране. Также
объявлен `BootReceiver`, который перезапускает службу после перезагрузки
телефона.

## 3. ЧТО СДЕЛАТЬ ПЕРЕД СБОРКОЙ (обязательно прочитать)

### Шаг 1 — создайте репозиторий на GitHub

Через мобильный веб-интерфейс GitHub: New repository → загрузите содержимое
этого zip-архива целиком (GitHub позволяет загружать файлы через
"Add file → Upload files" прямо с телефона; для множества файлов удобнее
через официальное приложение GitHub или Termux+git, если он у вас настроен).

### Шаг 2 — добавьте GitHub Secret (обязательно один)

Repository → **Settings** → **Secrets and variables** → **Actions** →
**New repository secret**:

- **Имя:** `GROQ_API_KEY`
- **Значение:** ваш ключ с https://console.groq.com/keys (бесплатная
  регистрация, обычным email — здесь никаких «корпоративных» ограничений нет)

Это единственный обязательный секрет. Без него приложение соберётся и
установится нормально, но умные ответы Groq не будут работать, пока вы не
вставите ключ вручную в Настройках приложения (это тоже сработает — ключ
можно задать и в самом приложении, без пересборки).

**Необязательные секреты** (только если хотите подписанный релизный APK
для публикации; без них соберётся debug-APK, который тоже прекрасно
устанавливается и работает на телефоне):
- `KEYSTORE_BASE64` — ваш `.keystore`/`.jks` файл, закодированный в base64
- `KEYSTORE_PASSWORD`
- `KEY_ALIAS`
- `KEY_PASSWORD`

### Шаг 3 — положите модель Vosk (ОБЯЗАТЕЛЬНО для рабочего wake-word и STT)

Это единственный файл, без которого голосовая активация и распознавание
команд не заработают (сама сборка APK при этом всё равно пройдёт успешно).

1. Скачайте `vosk-model-small-ru-0.22.zip` (~45 МБ) отсюда:
   https://alphacephei.com/vosk/models
   (ищите модель **vosk-model-small-ru-0.22** — компактная версия для
   мобильных устройств)
2. Распакуйте архив у себя (файловый менеджер на Android умеет
   распаковывать zip, либо Termux: `unzip vosk-model-small-ru-0.22.zip`)
3. Загрузите содержимое папки `vosk-model-small-ru-0.22/` (там будут
   подпапки `am/`, `conf/`, `graph/`, `ivector/` и т.д.) в репозиторий на
   GitHub, по пути `app/src/main/assets/vosk-model/`, **БЕЗ вложенной
   папки** — то есть в итоге должно получиться
   `app/src/main/assets/vosk-model/am/...`, а НЕ
   `app/src/main/assets/vosk-model/vosk-model-small-ru-0.22/am/...`

Подробная инструкция также лежит в `app/src/main/assets/vosk-model/README.md`
внутри архива.

### Шаг 4 — (опционально) своя иконка приложения

Положите файл `icon.png` (квадратный, рекомендуется 512×512, PNG) в папку
`icon/` в корне репозитория — то есть `icon/icon.png`.

Workflow сборки сам автоматически разложит эту иконку по всем нужным
размерам (`mipmap-mdpi` … `mipmap-xxxhdpi`) перед компиляцией — вам не
нужно ничего делать вручную. Если файл не добавить, приложение соберётся
с иконкой-заглушкой (тёмный фон + голубой глаз).

### Шаг 5 — запустите сборку

Вкладка **Actions** (в репозитории на GitHub) → **Build Evest Assistant
APK** → **Run workflow** → **Run workflow** (подтвердить). Сборка занимает
обычно 5–10 минут.

### Шаг 6 — скачайте готовый APK

После завершения workflow (зелёная галочка): откройте этот запуск → внизу
раздел **Artifacts** → скачайте `evest-assistant-debug-apk` (это ZIP,
внутри — сам `.apk`) или `evest-assistant-release-apk`, если настраивали
подпись.

## 4. Запуск на телефоне

1. Скачайте архив из Artifacts, распакуйте через файловый менеджер —
   внутри будет `.apk`.
2. Откройте `.apk` → Android спросит про установку "из неизвестных
   источников" (разрешить для приложения-файлового менеджера/браузера,
   через который открываете) → **Установить**.
3. Откройте **Evest Assistant** — при первом запуске приложение само
   покажет 3 экрана с объяснением и запросом разрешений:
   - микрофон + уведомления,
   - показ поверх экрана (для анимированного глаза),
   - исключение из оптимизации батареи (чтобы служба не засыпала в фоне).
4. Откройте **Настройки** (иконка шестерёнки) → вставьте **Groq API key**,
   если не задали его через GitHub Secret заранее → **Сохранить**.
5. Проверка wake word: скажите вслух «евестевень» — должен появиться
   анимированный глаз и подсказка «Слушаю...». Может потребоваться
   несколько попыток и чёткое произношение — офлайн-модель распознаёт
   изобретённое слово не идеально (см. раздел «Ограничения» ниже).
6. Пока распознавание настраивается — можно пользоваться кнопкой
   микрофона на главном экране вручную, все остальные функции работают
   независимо от wake-word.

### Отладка проблем

- **"Ключ не задан"** в статусе Groq → откройте настройки, вставьте ключ.
- **Wake word совсем не срабатывает** → откройте Настройки → включите
  «Показать лог» — если там есть сообщение "офлайн-модель распознавания
  речи не найдена", значит модель Vosk не была правильно добавлена перед
  сборкой (см. Шаг 3 выше — проверьте, что файлы лежат без вложенной папки).
- **Глаз не появляется поверх экрана** → Настройки Android → Приложения →
  Evest Assistant → Отображение поверх других приложений → включить вручную.
- **Служба останавливается через пару минут** → отключите оптимизацию
  батареи для приложения вручную: Настройки Android → Батарея →
  Evest Assistant → "Без ограничений".
- **Ложные срабатывания или слово не распознаётся** → можно
  скорректировать список принимаемых вариантов слова в исходном коде
  (`wakeword/WakeWordDetector.kt`, список `wakePhraseVariants`) и
  пересобрать — например, добавить варианты, которые чаще всего выдаёт
  распознавание на вашем голосе.

## 5. Ограничения (честно, без прикрас)

- **Wake-word через Vosk (а не специализированную модель)** — главный
  осознанный компромисс этой сборки. Причина выбора: единственные
  специализированные альтернативы (Porcupine, openWakeWord) либо требуют
  недоступной для многих регистрации, либо обучение модели через
  нестабильное окружение Google Colab, которое ломалось на конфликтах
  версий зависимостей несколько раз подряд при попытке обучить кастомную
  модель под «евестевень». Vosk работает сразу, без обучения и ключей, но
  распознаёт conкретное придуманное слово менее точно.
- **NFC для оплаты** — Android **не позволяет** сторонним приложениям
  программно включать/выключать NFC с Android 10+. Приложение честно об
  этом говорит и открывает экран настроек NFC, где нужно включить тумблер
  вручную один раз.
- **Пошаговая голосовая навигация внутри приложения** — Google не
  предоставляет публичный API для встраивания их голосовых поворотов в
  сторонние приложения. Реализовано так: Evest Assistant строит маршрут и
  запускает встроенную навигацию **в самом Google Maps**, где голос
  подсказок — это голос Google Maps, а не Евестевеня.
- **Управление чужим музыкальным плеером ("останови музыку")** — Android
  не даёт сторонним приложениям принудительно останавливать воспроизведение
  в другом приложении без привязки к его media-сессии. Честно сообщается
  пользователю, предлагается использовать кнопку паузы в шторке уведомлений.
- **Прямой звонок без открытия наборщика** — работает только если
  разрешение `CALL_PHONE` выдано; иначе открывается наборщик с готовым
  номером.
- **Отправка SMS без единого тапа** — не реализовано намеренно; открывается
  приложение сообщений с готовым текстом.
- **Автозапуск после перезагрузки** реализован через `BOOT_COMPLETED`, но
  некоторые производители (Xiaomi/MIUI, Huawei и т.п.) агрессивно
  ограничивают автозапуск сервисов — может потребоваться вручную разрешить
  автозапуск приложения в настройках производителя.

## 6. Улучшения после MVP

- Специализированная wake-word модель (Porcupine/openWakeWord/microWakeWord),
  если позже получится обойти регистрационные/технические барьеры —
  архитектура `WakeWordDetector.kt` спроектирована так, чтобы такую замену
  было легко подключить отдельным модулем позже, не трогая остальной код.
- Полноценные многодневные напоминания с локальной БД (Room) вместо
  делегирования системному будильнику.
- Поддержка умного дома (Google Home API / Matter).
- Виджет на рабочий стол с быстрым тапом активации.
- Настраиваемые сценарии-автоматизации.
- Полная история диалогов с поиском.
- Мультиязычность интерфейса и распознавания.
- Более точная офлайн-модель Vosk (large, ~1.5 ГБ) как опция в настройках
  для тех, кому критична точность больше размера APK.
